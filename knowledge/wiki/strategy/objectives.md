# Nido — Objectives ({period})

> **How to fill this out:** Objectives are the current period's measurable commitments. Different from vision (long-term) or strategy (how-to-win) — objectives are what you're trying to achieve in the next 90 days (or whatever your period is).
>
> Every spec, every plan, every implementation gets evaluated against: "Does this advance an objective? Which one?" If nothing, that's a red flag to discuss before proceeding.
>
> **Process:**
> 1. Pick a time horizon (90 days is a good default for early-stage)
> 2. Choose 3-5 objectives — no more
> 3. Each objective is an OUTCOME, not an output
> 4. Each objective is measurable, even imperfectly
> 5. Each objective connects to the vision
>
> 

---

## How to use this document

Every spec, plan, and implementation should reference an objective from this list. If a proposed feature doesn't advance any objective, that's a flag to discuss before proceeding.

The product agent and orchestrator both check this document at the start of `/spec`, `/plan`, and `/sprint` to evaluate whether new work is aligned.
