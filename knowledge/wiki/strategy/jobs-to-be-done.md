# Nido — Jobs to Be Done

> **How to fill this out:** Jobs are the underlying outcomes users hire Nido to deliver. Not features. Each job is phrased as: "When [situation], I want to [motivation], so I can [outcome]."
>
> **Process:**
> 1. Brainstorm 15-20 jobs without filtering. Think across all your personas.
> 2. Cluster them — group jobs that are variations of the same underlying need. You'll probably end up with 5-10 distinct clusters.
> 3. Categorize into wow / defense / nice-to-have based on importance.
>
> Examples to get you started:
> - "When I'm a new resident, I want to feel like I belong before I feel overwhelmed by logistics, so I can decide this community is home."
> - "When I'm running a community, I want to know which conflicts need my attention and which can wait, so I can intervene before things escalate without burning out on every small disagreement."
> - "When the community is making a decision, I want everyone's voice to be heard equally regardless of attendance, so the outcome reflects the community and not just whoever showed up."
