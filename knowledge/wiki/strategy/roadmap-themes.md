# Nido — Roadmap Themes

> **How to fill this out:** Roadmap themes are the strategic clusters that group features by intent — not a feature list. Themes give the product agent a way to evaluate "where does this feature fit?" before adding it to the backlog.
>
> **Process:**
> 1. Pick 3-5 themes for the next 12 months. No more.
> 2. Each theme has a "why this matters now"
> 3. Each theme has rough investment areas (not features yet)
> 4. Each theme has a sequence (when in the 12 months)
> 5. Explicitly name what's NOT a theme

## How features map to themes

When a new feature is proposed, identify which theme(s) it advances. Features that don't fit any theme should be discussed before being added — they're either an indicator of a missing theme or a sign of scope creep.

The product agent references this document during `/spec` and `/plan` to evaluate fit.
